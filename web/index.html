The Problem

Every CS student preparing for technical interviews has lived this moment.

"It's 11 PM. You've been staring at the same LeetCode problem for 45 minutes. You have two options: keep suffering with zero progress, or look up the solution, skim it, tell yourself you understand it, and move on. Sound familiar?"

Existing tools optimize for making you feel helped — not for making you actually learn. ChatGPT gives you a near-complete solution dressed up as a hint. Editorial solutions hand you the answer wholesale.

The gap between "I'm stuck" and "I looked up the answer" is where real understanding either forms or doesn't. AlgoSensei lives in that gap.

How It Works

Every student interaction goes through this pipeline.

1

Student sends a message or uploads a screenshot

Text description of where they're stuck, or a LeetCode problem image via Gemini Vision.

2

Mode detection routes to Explainer, Socratic, or Screenshot

A Gemini JSON call classifies intent: needs grounding, needs a hint, or uploaded an image.

3

Qdrant retrieves the top-4 most relevant knowledge chunks

The query is embedded locally with all-MiniLM-L6-v2 (pre-computed, zero API cost) and matched against the vector store via cosine similarity.

4

Gemini generates a calibrated hint using retrieved context

The system prompt encodes the hint level (1–3), negative constraints (no algorithm names, no code), and the retrieved chunks as grounding context.

5

Two-stage leakage gate validates every hint

Stage 1: regex catches algorithm names, code syntax, Big-O answers. Stage 2: semantic Gemini check for subtle leaks. Failed hints regenerate up to 3×.

6

Approved hint delivered with a 🔊 Listen button

Web Speech API speaks the hint at 0.88× speed — student can keep eyes on their code editor.

Example — Two Sum · Hint Level 1

Socratic mode · Level 1/3 · ✓ leakage check passed · Source: AlgoSensei KB — Arrays Hashing (intuition)
What do you notice about elements you've already scanned through? If you found the right answer at the very end of the array, what would you have needed to remember from earlier in the scan to recognize it immediately?

The system never said "hash map". The student figures that out themselves — which is the only way the intuition actually sticks.

Evaluation Metrics

Four measurable targets established in the project proposal.

>90%
Retrieval Recall @4
Correct concept in top 4 retrieved chunks
<5%
Hint Leakage Rate
Hints containing direct solution steps
>95%
Faithfulness Score
Claims grounded in retrieved context
>85%
Directional Accuracy
Hints pointing toward correct pattern

The Silent Failure — a fluent hint pointing toward the wrong algorithm — has a name, a detection mechanism, and a measurable target. The 110-pair synthetic evaluation dataset was built to find this failure, not hide it.

Run It Yourself

Three minutes to a working local instance.

# 1. Clone git clone https://github.com/adityaamitra/AlgoSensei cd AlgoSensei/algosensei_genai # 2. Install pip install -r requirements.txt # 3. Set API keys (.env file) cp .env.example .env # Add GEMINI_API_KEY, QDRANT_URL, QDRANT_API_KEY # 4. Build knowledge base in Qdrant (one-time, ~2 min) python scripts/setup_qdrant.py # 5. Generate evaluation data python scripts/generate_synthetic.py # 6. Launch streamlit run app.py

Free API keys: Gemini at aistudio.google.com · Qdrant at cloud.qdrant.io
