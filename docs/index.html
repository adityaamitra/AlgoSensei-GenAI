Why this exists

The tool I needed and didn't have

"It's 11 PM. You've been staring at the same LeetCode problem for 45 minutes. You have two options: keep suffering, or look up the solution, skim it, tell yourself you get it, and move on."

Every DSA prep tool either solves the problem for you or gives vague encouragement. ChatGPT hands you the answer. Editorials hand you the whole solution. There's no middle ground — until AlgoSensei.

AlgoSensei lives in the gap between "I'm stuck" and "I looked up the answer." It gives you the minimum hint needed to make progress. And it's architecturally enforced — the system cannot give a direct answer even if you ask repeatedly.

The pipeline

What happens on every request

Six steps from your question to a grounded, safety-checked hint.

1

You send a message or upload a screenshot

Text question, description of where you're stuck, or a LeetCode problem image.

2

Mode detector classifies your intent

Needs a concept explanation? A hint? Or uploaded an image? Routed to the right mode automatically.

3

Qdrant retrieves the top-4 knowledge chunks

Your query is embedded locally with all-MiniLM-L6-v2 and matched against 49 curated DSA knowledge chunks.

4

LLM generates a calibrated hint

The system prompt encodes the hint level (1–3) and hard constraints. Retrieved chunks are injected as grounding context.

5

Leakage gate validates the hint

Two-stage check: regex catches algorithm names and code; semantic check catches subtle leaks. Failed hints regenerate — you never see them.

6

Approved hint delivered with 🔊 Listen

You read the hint. One click speaks it aloud via the Web Speech API — eyes stay on your code.

See it in action

Two Sum — all three hint levels

Watch how the hints get progressively closer without ever naming the algorithm or writing code.

Hint 1 — Direction ✓ safe

"Take a tiny example — [2, 7, 11, 15] with target 9. As you examine each number, what value would you have needed to already encounter to reach the target? What would you need to keep track of to answer that instantly?"

Hint 2 — Structure ✓ safe

"As you walk through the array, what information would let you instantly tell whether the value you need has already been seen? If you could keep track of each number you've passed, how might you use that record?"

Hint 3 — Near-solution ✓ safe

"Scan the list just once. For each number, wonder whether you've already encountered a value that would add with it to reach the target. What would you record, and how could you quickly check if the partner is already recorded?"

The system never said "hash map". The student figures that out — which is the only way it actually sticks.

Self-host

Run it locally in 5 minutes

All you need is a free OpenRouter key and a free Qdrant Cloud account.

# Clone and install git clone https://github.com/adityaamitra/AlgoSensei cd AlgoSensei/algosensei_genai pip install -r requirements.txt # Add your keys cp .env.example .env # OPENROUTER_API_KEY=sk-or-v1-... # QDRANT_URL=https://your-cluster.qdrant.io # QDRANT_API_KEY=your-key # One-time setup — builds knowledge base in Qdrant python scripts/setup_qdrant.py # Launch streamlit run app.py

Free API keys: OpenRouter at openrouter.ai/keys · Qdrant at cloud.qdrant.io
