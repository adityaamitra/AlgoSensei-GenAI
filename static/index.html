Why this exists

The 11 PM problem

"You've been staring at the same LeetCode problem for 45 minutes. You have two options: keep suffering, or look up the solution, skim it, tell yourself you understand it, and move on."

Every DSA prep tool either solves the problem for you or gives vague encouragement. AlgoSensei does neither. It gives you the minimum hint needed to make progress — and it's architecturally enforced. The system cannot give a direct answer even if you ask.

The pipeline

What happens on every request

1
You send a message or upload a screenshot
Text question, description of where you're stuck, or a LeetCode problem image.
2
Mode detector classifies intent
Needs grounding? A hint? Or uploaded an image? Routed to the right mode automatically.
3
Qdrant retrieves the top-4 knowledge chunks
Query embedded locally with all-MiniLM-L6-v2 and matched against 49 curated DSA chunks.
4
LLM generates a calibrated hint
System prompt encodes hint level (1–3) and hard constraints. Retrieved chunks injected as grounding.
5
Leakage gate validates the hint
Regex + semantic check. Algorithm names, code syntax, Big-O reveals all blocked. Failed hints regenerate up to 3×.
6
Approved hint + 🔊 Listen button
You read the hint. One click speaks it aloud via Web Speech API at 0.88× speed.

See it in action

Two Sum — all three levels

Never names the algorithm. Never writes code. But by Level 3, you know exactly what to do.

Level 1 — Direction✓ safe
Take a tiny example — [2, 7, 11, 15] with target 9. As you examine each number one by one, what value would you have needed to already encounter to reach the target? What would you need to keep track of to answer that instantly?
Level 2 — Structure✓ safe
As you walk through the array, what information would let you instantly tell whether the value you need to reach the target has already been seen? If you could keep track of each number you've passed, how might you use that record?
Level 3 — Near-solution✓ safe
Scan the list just once. For each number, ask whether you've already encountered a value that would pair with it to reach the target. If yes — found it. If not — record this number so future checks can use it. What's the fastest way to do that lookup?

The system never said "hash map". You figure that out — which is the only way it sticks.
